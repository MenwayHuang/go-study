## 6.1 为什么需要服务发现？

```
❌ 硬编码服务地址:
   orderService.Call("192.168.1.100:8080")
   问题: 服务IP变了怎么办? 服务扩容了怎么办? 服务挂了怎么办?

✅ 服务发现:
   orderService.Call("order-service")  // 只需要服务名
   注册中心自动解析为可用的 IP:Port 列表
```

## 6.3 熔断器 — 防止级联雪崩

```
  服务A → 服务B → 服务C (挂了)

  没有熔断:
    服务A的请求全部堆积在B, B等待C超时
    → B也开始超时 → A也开始超时 → 全部崩溃 (级联雪崩)

  有熔断:
    C连续失败 → B的熔断器打开 → 直接返回错误(不再调用C)
    → A快速得到失败响应 → 不会堆积 → 系统稳定

  熔断器状态机:
    ┌──────┐  连续失败达阈值  ┌──────┐  超时后尝试  ┌──────┐
    │ 关闭  │───────────────→│ 打开  │─────────────→│ 半开  │
    │CLOSED │               │ OPEN  │              │HALF  │
    └──────┘               └──────┘              └──┬───┘
       ↑                                           │
       └────────── 探测成功,恢复 ──────────────────┘
                    探测失败 → 回到OPEN
```
