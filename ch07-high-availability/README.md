# 第7章：高可用设计

## 🎯 学习目标

掌握高可用系统的设计原则和实现技巧：重试策略、优雅降级、幂等设计、健康检查。

## 📚 高可用核心思想

```
高可用 = 系统在部分故障时仍能提供服务

可用性指标:
  99%    → 年停机 3.65天   (基本不可接受)
  99.9%  → 年停机 8.76小时  (普通系统)
  99.99% → 年停机 52.6分钟  (高可用系统)
  99.999%→ 年停机 5.26分钟  (金融级系统)
```

## 7.1 重试策略对比

```
固定间隔:     |--1s--|--1s--|--1s--|
指数退避:     |--1s--|---2s---|------4s------|
指数退避+抖动: |--0.8s--|---2.3s---|------3.7s------|

为什么需要抖动?
  1000个请求同时失败 → 1秒后同时重试 → 又全部失败 (重试风暴!)
  加了抖动后，重试时间分散开，避免风暴
```

## 7.3 幂等设计

```
什么是幂等?
  同一个操作执行一次和执行多次，效果相同

  幂等:   GET /user/1       (读取，天然幂等)
  幂等:   PUT /user/1 {}    (更新为固定值，幂等)
  非幂等: POST /orders      (创建，每次创建新订单)
  非幂等: POST /account/pay (扣款，每次都扣钱!)

  网络超时 → 客户端不知道服务端是否收到 → 重试 → 可能重复执行
  所以需要幂等设计来保证重复请求不会产生副作用
```
