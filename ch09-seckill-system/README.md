# 第9章：实战 — 高并发秒杀系统

## 🎯 学习目标

综合运用前8章所有知识，构建一个完整的高并发秒杀系统，并通过**内置压测工具**实际观测系统行为。

## 📚 秒杀系统的核心挑战

```
瞬时流量:   10万用户同时抢100个商品
读写比:     100:1 (大量查询库存，少量下单)
数据一致:   库存不能超卖 (100个商品不能卖出101个)
用户体验:   页面不能崩，要快速响应
```

## 🏗️ 系统架构 — 5层漏斗过滤

```
  用户请求 (10万QPS)
      │
  ┌───▼──────────────────────────────────┐
  │ 第1层: 令牌桶限流 (middleware.go)       │  500 req/s 放行，其余 429
  │ 生产对应: Nginx限流 / API Gateway      │
  └───┬──────────────────────────────────┘
      │ ~500 req/s
  ┌───▼──────────────────────────────────┐
  │ 第2层: 本地缓存售罄标记 (sync.Map)      │  纳秒级判断，售罄直接返回
  │ 生产对应: 进程内缓存 / 本地Redis        │
  └───┬──────────────────────────────────┘
      │ 未售罄的请求
  ┌───▼──────────────────────────────────┐
  │ 第3层: 幂等去重 (sync.Map LoadOrStore) │  同一用户只能抢一次
  │ 生产对应: Redis SETNX / 数据库唯一索引  │
  └───┬──────────────────────────────────┘
      │ 新用户的请求
  ┌───▼──────────────────────────────────┐
  │ 第4层: 原子减库存 (atomic.AddInt64)     │  无锁CAS操作，绝不超卖
  │ 生产对应: Redis DECR / DB乐观锁        │
  └───┬──────────────────────────────────┘
      │ 库存 >= 0
  ┌───▼──────────────────────────────────┐
  │ 第5层: 消息队列异步 (buffered channel) │  削峰填谷，快速返回用户
  │ 生产对应: Kafka / RabbitMQ / NATS     │
  └───┬──────────────────────────────────┘
      │
  ┌───▼──────────────────────────────────┐
  │ Worker Pool 消费 (5 goroutine)        │  异步创建订单
  │ 生产对应: 消费者组 + 写MySQL + 发通知   │
  └──────────────────────────────────────┘
```

## 🔍 观测要点（运行时注意看什么）

1. **服务端日志**: 看 Worker 创建订单的日志，注意是异步的
2. **压测报告**: 成功数是否 ≤ 库存数 → 验证防超卖
3. **失败原因分布**: 看哪一层拦截了最多请求 → 理解漏斗效果
4. **stats接口**: `rejected_by_sold` vs `rejected_by_stock` → 本地缓存 vs 原子操作
5. **订单数 vs 成功数**: 验证消息队列是否丢消息

## 💡 秒杀系统设计要点

### 1. 多级过滤，逐层减少请求
```
10万请求 → 限流(1000) → 本地缓存(500) → 幂等(400) → 原子库存(100) → MQ(100) → DB(100)
```

### 2. 库存预热
```
秒杀开始前: DB库存 → 加载到Redis → 加载到本地缓存
本项目中:   InitProduct() 直接加载到内存
```

### 3. 防止超卖的三道防线
```
防线1: atomic.AddInt64 原子减库存 → 主防线 (本项目)
防线2: 数据库乐观锁 UPDATE stock SET count=count-1 WHERE count>0 (生产补充)
防线3: 消息队列顺序消费 (本项目 Worker 串行处理)
```

### 4. 本项目 vs 生产环境对照

| 本项目(内存模拟) | 生产环境(真实组件) |
|----------------|------------------|
| `sync.Map` 售罄标记 | Redis/本地缓存 |
| `atomic.AddInt64` | Redis DECR |
| `chan *BuyRequest` | Kafka/RabbitMQ |
| `sync.Map` 去重 | Redis SETNX + TTL |
| `goroutine` Worker | 消费者组 + MySQL |
| `tokenBucket` 限流 | Nginx/API Gateway |
